# 🏋️ FitnessThing

FitnessThing — это Telegram Mini App для фитнеса с ИИ-ассистентом, агентским чатом и отслеживанием прогресса тренировок. Приложение разработано с подходом **Telegram-first** и оптимизировано для мобильного использования прямо в Telegram.

## ✨ Ключевые особенности

- 📱 **Telegram Mini App**: Нативная интеграция с Telegram, авторизация через Telegram
- 🤖 **Агентский ИИ-чат**: Интеллектуальный ассистент с инструментами для управления тренировками в реальном времени
- 📊 **Отслеживание прогресса**: Детальное логирование упражнений, подходов и весов
- 🎯 **Управление рутинами**: Создание и управление программами тренировок
- 💪 **База упражнений**: Обширная база данных упражнений с группировкой по мышечным группам
- 🧠 **Память пользователя**: ИИ запоминает предпочтения, травмы и особенности пользователя
- 📈 **Мониторинг**: Интеграция с Jaeger для distributed tracing

## 🏗️ Архитектура

### Технологический стек

#### Backend
- **Язык**: Go 1.23+
- **Framework**: gRPC с HTTP Gateway (grpc-gateway v2)
- **База данных**: PostgreSQL с pgx v5
- **Миграции**: Goose v3
- **Аутентификация**: Telegram Init Data
- **Трассировка**: Jaeger + OpenTracing
- **Хранилище файлов**: AWS S3 (или совместимое)
- **ИИ**: OpenAI API / OpenRouter (Claude, GPT и др.)
- **Конфигурация**: Viper с hot-reload

#### Frontend
- **Framework**: Next.js 15 с App Router и Turbopack
- **UI библиотека**: HeroUI (ex-NextUI)
- **Стили**: Tailwind CSS 4
- **Telegram SDK**: @telegram-apps/sdk-react
- **Анимации**: Framer Motion
- **Drag & Drop**: @dnd-kit
- **HTTP клиент**: Axios с автогенерацией типов

#### Infrastructure
- **Контейнеризация**: Docker + Docker Compose
- **Обратный прокси**: Nginx
- **Мониторинг**: Jaeger, Dozzle
- **База данных**: PostgreSQL

## 🤖 Агентский чат

Главная особенность FitnessThing — интеллектуальный агентский чат, который не просто отвечает на вопросы, а **активно управляет тренировками** пользователя. Агент имеет доступ к набору инструментов (tools) для взаимодействия с данными в реальном времени.

### Архитектура агента

Агент построен на основе LLM (OpenAI/Claude через OpenRouter) и использует function calling для выполнения действий. Каждое сообщение пользователя обрабатывается в контексте:
- Текущей тренировки (если открыта)
- Истории предыдущих тренировок
- Сохранённых фактов о пользователе (травмы, предпочтения, цели)
- Предыдущих сообщений в чате

### Инструменты агента

#### 📋 `list_muscle_groups`
Получает список всех доступных мышечных групп с их ID и названиями. Используется агентом для понимания, какие мышцы можно тренировать.

#### 🏃 `list_exercises`
Поиск упражнений по мышечным группам. Агент может отфильтровать упражнения по целевым мышцам и исключить уже добавленные в тренировку.

**Параметры:**
- `muscle_group_ids` — фильтр по мышечным группам
- `excluded_exercise_ids` — исключить упражнения (например, уже добавленные)

#### 📅 `get_workout_history`
Получает историю последних тренировок пользователя. Агент анализирует паттерны, нагрузку и прогресс.

**Параметры:**
- `workouts_limit` — количество тренировок (1-10)

#### 📊 `get_exercise_history`
Детальная история выполнения конкретного упражнения: веса, повторения, оценка нагрузки (power_rating 0-10).

**Параметры:**
- `exercise_id` — UUID упражнения
- `logs_limit` — количество записей (1-20)

#### 📝 `get_workout_plan`
Получает план текущей тренировки: список упражнений, запланированные подходы, уже выполненные сеты.

#### ➕ `add_exercises_to_workout`
Добавляет упражнения в текущую тренировку. Агент может самостоятельно составить тренировку по запросу пользователя.

**Параметры:**
- `exercise_ids` — массив UUID упражнений для добавления

#### ❌ `remove_exercise_logs_from_workout`
Удаляет упражнения из текущей тренировки.

**Параметры:**
- `exercise_log_ids` — массив UUID логов упражнений для удаления

#### 🔄 `replace_expected_sets`
Заменяет запланированные подходы для упражнения. Агент может скорректировать план на основе обратной связи пользователя.

**Параметры:**
- `exercise_log_id` — UUID лога упражнения
- `sets` — новый список подходов с весами и повторениями

#### 💾 `save_user_fact`
Сохраняет важную информацию о пользователе для будущих сессий. Агент автоматически запоминает:
- Травмы и ограничения
- Предпочтения в упражнениях
- Цели и режим тренировок
- Доступное оборудование

**Параметры:**
- `fact` — текст факта (до 500 символов)

### Примеры использования

**Пользователь:** "Добавь упражнения на грудь и трицепс"

**Агент:**
1. Вызывает `list_muscle_groups` для получения ID мышечных групп
2. Вызывает `list_exercises` с фильтром по груди и трицепсу
3. Вызывает `add_exercises_to_workout` с выбранными упражнениями
4. Отвечает пользователю с описанием добавленных упражнений

---

**Пользователь:** "У меня болит плечо, замени жим штанги на что-то другое"

**Агент:**
1. Вызывает `save_user_fact` для запоминания проблемы с плечом
2. Вызывает `get_workout_plan` для понимания текущей тренировки
3. Вызывает `list_exercises` для поиска альтернатив
4. Вызывает `remove_exercise_logs_from_workout` для удаления жима
5. Вызывает `add_exercises_to_workout` для добавления альтернативы
6. Отвечает с рекомендациями и объяснением выбора

### Квоты и лимиты

- Дневной лимит сообщений на пользователя (настраивается)
- Резервирование квоты перед обработкой запроса
- Возврат неиспользованной квоты при ошибках
